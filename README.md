# YOLO-TFLite-Android

**Android YOLO Object Detection Inference Framework**

A high-performance Android inference app built on [LiteRT 2.1.4](https://ai.google.dev/edge/litert) (formerly TensorFlow Lite), supporting **YOLO numeric series** (YOLOv5 / v8 / v10 / v11 / v12 / YOLO26), NPU / GPU / CPU accelerators, 5 vision tasks, and 4 detection modes.

## Features Overview

| Feature | Description |
|---------|-------------|
| **Multi-task** | Object Detection, Keypoint Detection, Instance Segmentation, Image Classification, Oriented BBox Detection |
| **Multi-accelerator** | NPU (Qualcomm Hexagon / MediaTek), GPU (OpenCL / OpenGL), CPU (XNNPack) |
| **Zero-copy pipeline** | GL Compute Shader + BLOB AHardwareBuffer direct to NPU (MTK/Tensor only) |
| **GPU preprocessing** | OpenGL ES 3.0 letterbox + normalization, offloading CPU workload |
| **AOT pre-compilation** | Ahead-of-time NPU compilation, skipping JIT delay for optimal first-run performance |
| **Dual inference backends** | Native C++ JNI (recommended) and Java/Kotlin, switchable |
| **Multi-precision models** | INT8 / FLOAT16 / FLOAT32 |
| **Flexible model sources** | Built-in assets / Local import (directory or ZIP) / Remote download, no conflicts |
| **Material 3 UI** | Built with Jetpack Compose |

> **⚠️ Important**: Only **YOLO26 series** supports End-to-End inference mode, and End-to-End mode **only supports CPU**, not GPU / NPU acceleration. Other YOLO series only support the traditional mode.

## Quick Start

### Option 1: Android Studio (Recommended)

1. Install [the latest Android Studio](https://developer.android.com/studio) and open this project
2. Configure paths and target SoC in `local.properties`:

```properties
# Usually auto-generated by Android Studio
sdk.dir=/path/to/android-sdk
# Optional, uses Android Studio's built-in JDK if not set
org.gradle.java.home=/path/to/jdk-21
# Optional: sm8450 / sm8550 / sm8650 / sm8750 / sm8850 / qualcomm (all Qualcomm, default) / mediatek
targetSoc=sm8850
```

3. **(Required only for Qualcomm NPU)** Download QNN runtime libraries (~1-2 GB):

```bash
cd litert_npu_runtime_libraries
# Windows
.\fetch_qualcomm_library.ps1
# Linux / macOS
./fetch_qualcomm_library.sh
```

> Skip this step if only using GPU / CPU inference. Edit the `QnnVersions` array in the script to download only specific SoC runtimes.

4. Click Run. Android Studio will automatically install the base APK + the corresponding QNN split APK.

---

### Option 2: Command Line

The project provides isolated environment scripts. All dependencies are downloaded to `.deps/` without affecting the host system:

```bash
# Windows (choose one)
.\setup-isolated-env-aria2.ps1    # aria2 multi-threaded, recommended
.\setup-isolated-env-surge.ps1    # Using surge download tool

# Linux / macOS
chmod +x setup-isolated-env.sh && ./setup-isolated-env.sh
```

After setup, download Qualcomm QNN runtimes (same steps as above), then build and install:

```bash
# Windows
.\gradlew.bat installLocalDebug

# Linux / macOS
./gradlew installLocalDebug
```

> **Tips**: Connect your phone via USB with **USB debugging** or **wireless debugging** enabled. Verify with `adb devices`. `installLocalDebug` automatically calls `adb install-multiple` to install the base APK + runtime split APK.

> The first build will download Gradle, LiteRT C++ SDK, and other dependencies automatically. If you encounter CMake / C++ compilation errors, try deleting `detector/.cxx/` and rebuilding.
>
> If you have a local proxy enabled, uncomment and fill in the proxy configuration in `gradle.properties`:
> ```properties
> systemProp.http.proxyHost=127.0.0.1
> systemProp.http.proxyPort=Your Proxy Port
> systemProp.https.proxyHost=127.0.0.1
> systemProp.https.proxyPort=Your Proxy Port
> ```

## Supported Vision Tasks & Detection Modes

### Vision Tasks

| Task Type | Default Input Size |
|-----------|-------------------|
| Object Detection | 640×640 |
| Keypoint Detection | 640×640 |
| Instance Segmentation | 640×640 |

| Oriented BBox Detection | 1024×1024 |

> Custom-trained models **must be trained and exported at the default input size** for the corresponding task.

|------|-------------|

## Accelerators & Pipelines

### Accelerator Compatibility

| Model Precision | CPU | GPU | NPU |
|-----------------|:---:|:---:|:---:|
| INT8 | ✅ | ✅ | ❌ |
| FLOAT16 | ✅ | ✅ | ✅ |
| FLOAT32 | ✅ | ✅ | ✅ (internally converted to FP16) |
| End-to-End (YOLO26 only) | ✅ | ❌ | ❌ |

> FLOAT16 models are recommended for NPU. FLOAT32 is internally converted to FLOAT16 on NPU with equivalent accuracy and size — exporting as FLOAT16 directly reduces file size.

### Supported NPU Devices

| Platform | SoC | Chip | Minimum OS |
|----------|-----|------|-----------|
| **Qualcomm** | SM8450 | Snapdragon 8 Gen 1 | Android 12+ |
| | SM8550 | Snapdragon 8 Gen 2 | Android 12+ |
| | SM8650 | Snapdragon 8 Gen 3 | Android 12+ |
| | SM8750 | Snapdragon 8 Elite | Android 12+ |
| | SM8850 | Snapdragon 8 Elite Gen 5 | Android 12+ |
| **MediaTek** | MT6878 | Dimensity 7300 | **Android 15 only** |
| | MT6897 | Dimensity 8300 | **Android 15 only** |
| | MT6983 | Dimensity 9000 | **Android 15 only** |
| | MT6985 | Dimensity 9200 | **Android 15 only** |
| | MT6989 | Dimensity 9300 | **Android 15 only** |
| | MT6991 | Dimensity 9400 | **Android 15 only** |
| **Google** | Tensor | Pixel 6/7 | ⚠️ Implemented, not tested on device |

### Pipeline Strategies

The app automatically selects based on hardware capabilities, or you can switch manually:

| Strategy | Display Name | Description | Platform |
|----------|-------------|-------------|----------|
| GL_ZEROCOPY | Fastest | GPU → NPU zero-copy, lowest latency | MTK / Tensor NPU |
| GL_TRANSIT | Accelerated | GPU preprocessing + letterbox normalization | All platforms |
| CPU_PIPELINE | Compatible | Pure CPU preprocessing, most stable | All platforms |

> Qualcomm NPU uses FastRPC buffers and does not support the GL zero-copy path. The highest available option is GL_TRANSIT.

## Model Management

### Getting Models

This project **does not include model files**. Three sources are supported (no conflicts between them):

| Source | Description |
|--------|-------------|
| **Built-in** | Place `.tflite` models in `app/src/main/assets/` following the directory convention, bundled with APK |
| **Local Import** | Select a model directory or ZIP archive via in-app SAF |
| **Remote Download** | Download from an HTTP server within the app, supports resume and auto-extraction |

### Directory Structure Convention

```
{taskType}/tflite/{packageName}/
├── label.txt               # Label file (required)
└── models/
    └── {size}/             # n / s / m / l / x
        ├── model.tflite
        └── aot/            # AOT pre-compiled models (optional)
            ├── model_Qualcomm_SM8650.tflite
            └── ...
```

- `taskType`: `detection` / `keypoint` / `segmentation` / `classification` / `oriented_bbox`
- `packageName`: Format `{base}[-{task}]`, e.g., `yolo26n`, `yolo26-pose`
- `size`: `n`=Nano, `s`=Small, `m`=Medium, `l`=Large, `x`=XLarge

Built-in / Local / Remote models with the same `packageName` are isolated by composite keys — no conflicts.

## TFLite Model Conversion

Use [Ultralytics](https://github.com/ultralytics/ultralytics) to export. Key parameters:

| Parameter | Description |
|-----------|-------------|
| `format="tflite"` | Export format |
| `imgsz` | Input size, must match the task's default size |
| `half=True` | Export FLOAT16 |
| `int8=True` | Export INT8 (requires `data` for calibration set) |
| `end2end=False` | **Must be set for YOLO26**, otherwise exports end-to-end mode (CPU only) |

```python
from ultralytics import YOLO

model = YOLO("yolo26n.pt")

# FLOAT16, supports GPU / NPU (recommended)
model.export(format="tflite", imgsz=640, end2end=False, half=True)

# INT8 quantization
model.export(format="tflite", imgsz=640, end2end=False, int8=True, data="coco8.yaml")

# End-to-end mode (CPU only)
model.export(format="tflite", imgsz=640)
```

**Different task types:**

```python
model = YOLO("yolo26n-pose.pt")   # Keypoint, imgsz=640
model = YOLO("yolo26n-seg.pt")    # Segmentation, imgsz=640
model = YOLO("yolo26n-cls.pt")    # Classification, imgsz=224, no end2end needed
model = YOLO("yolo26n-obb.pt")    # Oriented bbox, imgsz=1024
```

**Output Shape Reference:**

| Mode | Output Shape | Example |
|------|-------------|---------|
| End-to-End (with NMS) | `(1, max_det, features)`, features < max_det | `(1, 300, 6)` |
| Traditional (post-processing required) | `(1, channels, candidates)`, candidates > channels | `(1, 84, 8400)` |

The app auto-detects the inference mode from the output shape — no manual configuration needed.

## AOT Pre-compilation

AOT pre-compiles NPU models ahead of time, eliminating first-inference JIT delay.

### Compilation Script

```bash
pip install "ai-edge-litert>=2.1.4" "ai-edge-litert-sdk-qualcomm>=0.2.0"

# Compile all built-in models
python aot_compile.py

# Specify SoC
python aot_compile.py --soc SM8650

# Specify model file
python aot_compile.py --model path/to/model.tflite --soc SM8650
```

### Naming Convention

```
{original_model_name}_Qualcomm_{SoC_model}.tflite
# Example: yolo26s_float16-no-nms_Qualcomm_SM8650.tflite
```

Place AOT models in the corresponding `aot/` subdirectory. The app automatically matches and prioritizes AOT models based on the device's SoC at startup, falling back to JIT if no match is found.

## Inference Configuration

### Accelerator Selection Guide

| Scenario | Recommended Configuration |
|----------|--------------------------|
| Best performance | Native + NPU + FLOAT16 traditional mode |
| GPU acceleration | Native + GPU + INT8 traditional mode |
| Maximum compatibility | Native + CPU + any model |
| Highest accuracy | Native + CPU + FLOAT32 |

### Inference Backends

| Backend | Recommended | Description |
|---------|:-----------:|-------------|
| LiteRT Native (C++ JNI) | ✅ | Supports JIT cache + AOT + zero-copy |
| LiteRT Java (Kotlin) | — | Fallback, supports AOT only |

| CMake | 3.22.1+ |

### LiteRT C++ SDK

Automatically downloaded from [GitHub Releases](https://github.com/google-ai-edge/LiteRT/releases) and configured during the first Gradle sync. If the download fails, manually download `litert_cc_sdk.zip`, extract it to `detector/src/main/cpp/litert_cc_sdk/`, delete the `detector/.cxx/` cache, and re-sync.

### NPU Runtime (targetSoc Configuration)

`targetSoc` controls which NPU runtime modules are packaged. Precise configuration reduces APK size (each runtime is ~10-20 MB):

| Value | Packaged Content |
|-------|-----------------|
| `qualcomm` (default) | All Qualcomm runtimes (v69-v81) |
| `sm8450` / `sm8550` / `sm8650` / `sm8750` / `sm8850` | Corresponding single SoC runtime |
| `mediatek` | MediaTek runtime (requires Android 15) |

## Known Limitations

| Limitation | Description |
|------------|-------------|
| End-to-End CPU only | NMS op is not supported by GPU / NPU; only YOLO26 series has this mode |
| Zero-copy MTK / Tensor only | Qualcomm NPU uses FastRPC buffers, cannot achieve GL → NPU zero-copy |
| MTK NPU Android 15 only | Official examples declare Android 15 only; tested that Android 16 is not supported |
| Google Tensor NPU untested | Runtime module is reserved, not verified on real devices |
| arm64-v8a only | No 32-bit architecture support |
| minSdk 31 | Android 11 and below not supported |
| Remote download is dev-only | Default server address is LAN, requires manual configuration |
| Other YOLO series not fully tested | YOLOv5/v8/v10/v11 are theoretically compatible, may need post-processing adjustments |
